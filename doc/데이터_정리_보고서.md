## 📊 1. 데이터 개요 슬라이드 (Data Overview)

### **기본 정보**
- **데이터 출처**: 하나투어 내부 VoC 시스템(리뷰), 공공 데이터 포털(항공 통계), 하나투어 상품 마스터 DB
- **수집 방법**: CSV 형식의 정형/비정형 데이터 통합 (3-Way Data Blending)
- **수집 기간**: 2011년 09월 ~ 2024년 03월 (주요 분석 대상: 2018년 ~ 2024년)
- **분석 대상**: 베트남(다낭, 나트랑) 및 싱가포르 노선 패키지/에어텔 이용 고객

### **데이터 규모**
- **전체 행(Records)**: 11,225건 (통합 리뷰 데이터 기준)
- **전체 열(Columns)**: 18개

### **주요 컬럼 정의**
| 컬럼명     | 설명                                        | 비즈니스 의미                                |
| :--------- | :------------------------------------------ | :------------------------------------------- |
| `대상도시` | 여행 목적지 (다낭, 나트랑, 싱가포르)        | 분석 세그먼트 분류 기준                      |
| `평점`     | 고객 만족도 점수 (1.0 ~ 5.0)                | 서비스 품질 평가의 핵심 종속 변수            |
| `내용`     | 고객이 작성한 서술형 후기                   | 텍스트 마이닝 및 감성 분석의 원천 데이터     |
| `작성일`   | 리뷰 등록 시간 정보                         | 시계열 트렌드 및 계절성(Seasonality) 분석용  |
| `동행`     | 함께 여행한 인원 유형 (가족, 커플, 친구 등) | 세그먼트별 페인포인트 식별 변수              |
| `성인가격` | 1인당 상품 판매가                           | 고객의 기대 심리 및 기대 불일치 분석용       |
| `쇼핑횟수` | 일정 내 포함된 의무 쇼핑 횟수               | 가격-품질 트레이드오프 분석의 핵심 독립 변수 |

## 🛠️ 2. 전처리 요약 슬라이드 (Data Preprocessing)

### **결측값 및 이상값 처리**
- **결측값 처리**: 
    - `성인가격`, `쇼핑횟수`: 전처리 과정에서 `pd.to_numeric(errors='coerce')` 적용 후, 누락값은 `0` 또는 전체 평균값으로 대체(Imputation). (전체 결측률 약 2.5% 미만)
    - `내용`: 내용이 없는 리뷰는 분석 대상에서 제외.
- **이상값 처리**:
    - **리뷰 길이**: 공백만 있는 무의미한 리뷰 및 시스템 오류로 인한 중복 데이터 제거 (약 120건 제거).
    - **가이드 비용 및 기타 수치**: 비정상적으로 높은 가격(Dummy Data)을 사전에 정의된 임계값(Threshold) 기준으로 필터링.

### **최종 데이터 현황**
- **분석 사용 데이터**: 11,225행 / 20개 열 (파생 변수 포함)

### **주요 파생 변수 생성 (Feature Engineering)**
1. **`리뷰길이`**: 고객의 불만 강도를 정량화하기 위해 `내용`의 텍스트 글자 수 산출.
2. **`평점대`**: 평점 데이터를 '고위험(1~3점)', '우수(4~5점)' 등 범주형 그룹으로 변환.
3. **`IATA코드`**: 도시명을 공공 항공 데이터와 연동하기 위해 'DAD', 'CXR', 'SIN' 코드로 매핑.
4. **`고위험여부`**: '평점 3.0 이하' 이면서 '리뷰 500자 이상'인 리뷰를 리얼타임 리스크 지표로 정의.

## ⚠️ 3. 데이터 한계점 (Data Limitations)

- **경쟁사 데이터 부재**: 하나투어 단독 데이터로 분석되어, 타 여행사(모두투어, 노랑풍선 등)와의 상대적 시장 만족도 비교에 한계가 있음.
- **감성 분석의 한계**: NLP 라이브러리가 '반어법'이나 '특수 전문 용어'를 완벽히 해석하지 못할 수 있어, 수치 기반 감성이 실제 맥락과 미세한 차이가 있을 수 있음.
