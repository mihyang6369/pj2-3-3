# [데이터 분석] 하나투어 해외여행지 데이터 정제 보고서

본 보고서는 하나투어의 해외 여행지(다낭, 나트랑, 싱가포르) 리뷰 및 상품 데이터를 분석하기 위해 수행된 **데이터 정제(Data Refinement) 및 전처리(Preprocessing)** 과정을 상세히 기록한 문서입니다.

---

## 1. 데이터 수집 및 원천 데이터 구성 (Data Acquisition)

분석에 사용된 데이터는 총 3가지 범주, 10여 개의 CSV 파일로 구성되어 있습니다.

- **리뷰 데이터 (`hanatour_reviews.csv`)**: 고객이 작성한 실제 여행 후기, 평점, 작성일 등을 포함합니다.
- **항공 실적 데이터 (`processed_aviation_performance.csv`)**: 각 도시별(IATA 코드 기준) 월별 항공 운항 및 승객 수 실적입니다.
- **도시별 통합 상품 데이터**: 다낭, 나트랑, 싱가포르의 상품 정보(가격, 쇼핑 횟수, 가이드 유무 등)를 포함하며, 상품군(패키지, 에어텔, 투어/티켓)별로 분리되어 있습니다.

---

## 2. 데이터 통합 및 병합 (Data Integration & Merging)

여러 소스로 분산된 데이터를 하나의 통합 데이터프레임으로 합치는 과정입니다.

1.  **상품 마스터(Product Master) 생성**: 
    - 각 도시/상품군별 CSV에서 `대표상품코드`를 키(Key)로 하여 중복을 제거한 통합 상품 정보를 생성합니다.
2.  **리뷰-상품 병합**: 
    - `리뷰.상품코드`와 `상품마스터.대표상품코드`를 매칭하여 리뷰 데이터에 상품의 가격, 쇼핑 횟수 정보를 결합합니다.
3.  **항공 실적 결합**: 
    - `대상도시`를 `IATA 코드`(DAD, CXR, SIN)로 변환한 후, `월` 정보를 기준으로 항공 실적(평균 항공객수)을 병합합니다.

---

## 3. 주요 정제 및 예외 처리 (Data Cleaning)

데이터의 일관성을 확보하기 위해 다음과 같은 정제 작업을 수행했습니다.

- **컬럼명 표준화**: 
    - 상품 유형에 따라 `성인가격` 또는 `정상가격`으로 혼용되는 컬럼을 `성인가격`으로 통일했습니다.
- **결측치(Null) 처리**: 
    - 상품군 정보가 없는 경우 `정보없음`으로 채워 분석에서 제외되지 않도록 했습니다.
    - 가격이나 쇼핑 횟수가 누락된 경우 `0`으로 대체하여 수치 계산 시 오류를 방지했습니다.
- **공백 제거**: 
    - CSV 로드 시 컬럼명 전후에 포함될 수 있는 불필요한 공백을 `str.strip()`으로 제거했습니다.

---

## 4. 데이터 타입 변환 (Type Casting)

분석 도구(Plotly, Pandas)와의 호환성을 위해 데이터 타입을 교정했습니다.

- **날짜 데이터**: `작성일` 컬럼을 `datetime` 객체로 변환하여 연도별/월별 추이 분석이 가능하게 했습니다.
- **수치 데이터**: 문자열로 인식될 수 있는 `성인가격`, `쇼핑 횟수`, `평점` 등을 `numeric` 타입으로 강제 변환했습니다.
- **인코딩**: 모든 한글 파일은 `utf-8-sig` 형식을 사용하여 한글 깨짐 현상을 방지했습니다.

---

## 5. 파생 변수 생성 (Feature Engineering)

단순 데이터를 분석 가능한 지표로 전환하기 위해 새로운 변수를 생성했습니다.

- **평점대**: `평점`을 기반으로 '4점대', '5점대'와 같은 범주형 변수를 생성하여 만족도 그룹 분석에 활용했습니다.
- **IATA 코드**: 도시명을 항공 통계와 연동하기 위해 국제항공운송협회(IATA) 코드로 매핑했습니다.
- **월간 평균 항공객수**: 특정 도시의 특정 월에 해당하는 항공 수요를 산출하여 리뷰 만족도와의 상관관계를 분석할 기초를 마련했습니다.

---

## 6. 결론 및 기대 효과

이러한 정제 과정을 통해 확보된 **Clean Data**는 다음과 같은 효과를 제공합니다.

1.  **분석의 신뢰성**: 결측치와 이상치가 제거되어 정확한 통계 산출이 가능합니다.
2.  **시각화의 일관성**: 데이터 타입이 통일되어 차트 생성 시 발생하는 런타임 에러를 방지합니다.
3.  **확장성**: 추후 다른 도시의 데이터가 추가되더라도 동일한 정제 로직을 적용할 수 있는 구조(Modular Code)를 갖추었습니다.
